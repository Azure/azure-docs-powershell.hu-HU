# Enable-AzContextAutosave

## Áttekintés
Az Azure-környezetek az aktív előfizetést jelképező PowerShell-objektumok, valamint az Azure Cloud-hoz való csatlakozáshoz szükséges hitelesítési adatok. Az Azure-környezetek használatakor az Azure PowerShell-nek nem kell minden alkalommal újra hitelesítenie a fiókját az előfizetések váltása során. További tudnivalókat az [Azure PowerShell környezet-objektumai](https://docs.microsoft.com/powershell/azure/context-persistence)című témakörben találhat.

Ez a parancsmag lehetővé teszi az Azure-környezet adatainak mentését és automatikus betöltését PowerShell-folyamat indításakor. Új ablak megnyitásakor például

## SZINTAXISA

```
Enable-AzContextAutosave [-Scope <ContextModificationScope>] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Leírás

Lehetővé teszi az Azure-környezet adatainak mentését és automatikus betöltését a PowerShell-folyamat indításakor. A környezet a környezetet érintő bármely parancsmag végrehajtásának végén tárolódik. Például bármely profil parancsmag. Ha a felhasználó hitelesítését használja, a tokenek bármelyik parancsmag futtatása során frissíthetők.

## Példák

### Példa 1: az aktuális felhasználó automatikus mentési hitelesítő adatainak engedélyezése

A hitelesítő adatok automatikus mentésének bekapcsolása az aktuális felhasználónál Minden alkalommal, amikor egy PowerShell-ablak van megnyitva, a jelenlegi környezete bejelentkezés nélkül emlékezetbe kerül.

```powershell
Enable-AzContextAutosave
```

### 2. példa

Engedélyezze az Azure hitelesítő adatait, fiókját és előfizetési adatait, hogy a PowerShell-munkamenet PowerShell-ablakának megnyitásakor a program mentse és automatikusan betöltse a fájlt. autogenerated

