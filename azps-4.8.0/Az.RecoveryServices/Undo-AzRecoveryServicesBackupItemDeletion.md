# Undo-AzRecoveryServicesBackupItemDeletion

## Áttekintés
Ha egy biztonságimásolat-elem törlődik, és egy finoman törölt állapotban van, ez a parancs visszaállítja azt az állapotot, ahol az adat örökre megmarad 

## SZINTAXISA

```
Undo-AzRecoveryServicesBackupItemDeletion [-Item] <ItemBase> [-Force] [-VaultId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Leírás
A Undo-AzRecoveryServicesBackupItemDeletion parancsmag egy olyan állapotra állítja vissza a finoman törölt elemeket, amelyben a védelem le van állítva, az adatmegőrzés azonban örökké megmarad.

## Példák

### Példa 1
```powershell
PS C:\> $Cont = Get-AzRecoveryServicesBackupContainer -ContainerType AzureVM
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM 
PS C:\> Disable-AzRecoveryServicesBackupProtection -Item $PI[0] -RemoveRecoveryPoints
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM | Where-Object {$_.DeleteState -eq "ToBeDeleted"}
PS C:\> Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0]
```

Az első parancs a biztonságimásolat-tárolók tömbjét kapja, majd a $Cont tömbben tárolja.
A második parancs az első tároló elemnek megfelelő biztonsági másolatot kapja, majd a $PI változóban tárolja.
A harmadik parancs letiltja a biztonsági mentést $PI 0 elemen, \[ \] és egy softdeleted-állapotba helyezi az elemet.
A negyedik parancs a softdeleted-állapotot tartalmazó elemet kapja.
Az utolsó parancs a softdeleted VM-et egy olyan államba hozza létre, ahol a védelem megszűnt, az adatmegőrzés azonban örökké megmarad.

### 2. példa

Újrahidratálja a finoman törölt elemeket. autogenerated

```powershell
<!-- Aladdin Generated Example --> 
Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0] -VaultId $vault.ID
```
