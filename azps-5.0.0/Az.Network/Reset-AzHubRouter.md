# Reset-AzHubRouter

## Áttekintés
Visszaállítja egy VirtualHub-erőforrás RoutingState.

## SZINTAXISA

### ByVirtualHubName (alapértelmezett)
```
Reset-AzHubRouter -ResourceGroupName <String> -Name <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubResourceId
```
Reset-AzHubRouter -ResourceId <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubObject
```
Reset-AzHubRouter -InputObject <PSVirtualHub> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Leírás
Egy meglévő VirtualHub-erőforrás útválasztási állapotának visszaállítása csak akkor, ha a virtuális hub útválasztási állapota nincs kiépítve.

## Példák

### Példa 1

```powershell
PS C:\> Reset-AzHubRouter -ResourceGroupName "testRG" -Name "westushub"
```

Állítsa alaphelyzetbe a virtuális hub útválasztási állapotát a ResourceGroupName és a ResourceName segítségével.

### 2. példa

```powershell
PS C:\> Reset-AzHubRouter -ResourceId "/subscriptions/testSub/resourceGroups/testRG/providers/Microsoft.Network/virtualHubs/westushub"
```

Állítsa alaphelyzetbe a virtuális hub útválasztási állapotát a ResourceId.

### 3. példa

```powershell
PS C:\> Reset-AzHubRouter -InputObject $virtualHub
```

A virtuális hub útválasztási állapotának visszaállítása beviteli objektum használatával. A beviteli objektum PSVirtualHub típusú.

### 4. példa

```powershell
PS C:\> Get-AzVirtualHub -ResourceGroupName "testRG" -Name "westushub" | Reset-AzHubRouter
```

A meglévő virtuális hub-objektumok beolvashatók, majd a AzHubRouter alaphelyzetbe kerülnek a beviteli objektumként.
