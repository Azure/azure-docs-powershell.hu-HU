# Start-AzBatchPoolResize

## SYNOPSIS
Kezd átméretezni egy készletet.

## SZINTAXIS

```
Start-AzBatchPoolResize [-Id] <String> [-TargetDedicatedComputeNodes <Int32>]
 [-TargetLowPriorityComputeNodes <Int32>] [-ResizeTimeout <TimeSpan>]
 [-ComputeNodeDeallocationOption <ComputeNodeDeallocationOption>] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## LEÍRÁS
A **Start-AzBatchPoolResize** parancsmag elindít egy Azure Batch-átméretezési műveletet egy készleten.

## PÉLDÁK

### 1. példa: Készlet átméretezése 12 csomópontra
```
PS C:\>Start-AzBatchPoolResize -Id "ContosoPool06" -TargetDedicatedComputeNodes 12 -BatchContext $Context
```

Ez a parancs egy átméretezési műveletet kezd a ContosoPool06 azonosítójú készleten.
A művelet célja 12 dedikált számítási csomópont.
A Get-AzBatchAccountKey parancsmag használatával kontextust rendelhet a $Context változóhoz.

### 2. példa: Készlet átméretezése áthelyezési beállítással
```
PS C:\>Get-AzBatchPool -Id "ContosoPool06" -BatchContext $Context | Start-AzBatchPoolResize -TargetDedicatedComputeNodes 5 -ResizeTimeout ([TimeSpan]::FromHours(1)) -ComputeNodeDeallocationOption ([Microsoft.Azure.Batch.Common.ComputeNodeDeallocationOption]::Terminate) -BatchContext $Context
```

Ez a parancsmag átméretez egy készletet öt dedikált számítási csomópontra.
A parancs a ContosoPool06 azonosítót használó készletet a Get-AzBatchPool parancsmag használatával kapja meg.
A parancs a folyamat műveleti operátorával átadja az objektumot az aktuális parancsmagnak.
A parancs átméretezést kezd a készleten.
A cél öt dedikált számítási csomópont.
A parancs egyórás időkorrektot ad meg.
A parancs a Terminate (Lezárás) elosztási lehetőséget adja meg.

## PARAMETERS

### -BatchContext
Azt a **BatchAccountContext példányt** adja meg, amely a parancsmagot használja a Batch szolgáltatáshoz való interakcióhoz.
Ha a Get-AzBatchAccount parancsmaggal szerezze be a BatchAccountContext parancsmagot, akkor a Batch szolgáltatás használata során az Azure Active Directory-hitelesítést fogja használni a rendszer. Ha ehelyett megosztott kulcsú hitelesítést használ, a Get-AzBatchAccountKey parancsmaggal töltse ki a BatchAccountContext objektumot a hozzáférési kulcsokkal. Megosztott kulcs hitelesítése esetén a rendszer alapértelmezés szerint az elsődleges hozzáférési kulcsot használja.
