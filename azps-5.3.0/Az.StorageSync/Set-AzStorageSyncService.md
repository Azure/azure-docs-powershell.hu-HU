# Set-AzStorageSyncService

## SYNOPSIS
Ez a parancs beállítja a tárterület-szinkronizálási szolgáltatást egy erőforráscsoportban.

## SZINTAXIS

```
Set-AzStorageSyncService [-ResourceGroupName] <String> [-Name] <String> [-Location] <String> [-Tag <Hashtable>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## LEÍRÁS
A tárterület-szinkronizálási szolgáltatás az Azure File Sync legfelső szintű erőforrása. Ez a parancs beállítja a tárterület-szinkronizálási szolgáltatást egy erőforráscsoportban. Azt javasoljuk, hogy hozzon létre annyi tárterület-szinkronizálási szolgáltatást, amennyit feltétlenül szükséges a szervezet kiszolgálói csoportjainak megkülönböztetésére. A tárterület-szinkronizálási szolgáltatás szinkronizálási csoportokat tartalmaz, és célként is működik a kiszolgálók regisztrálására. Egy kiszolgáló csak egyetlen tárterület-szinkronizálási szolgáltatásba regisztrálható. Ha a kiszolgálóknak részt kell venniük ugyanazon fájlkészlet szinkronizálásában, regisztrálja őket ugyanannak a tárhelyszinkronizálási szolgáltatásnak.

## PÉLDÁK

### 1. példa
```powershell
PS C:\> Set-AzStorageSyncService -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -IncomingTrafficPolicy "AllowAllTraffic"
```

Ez a parancs beállít egy tárterület-szinkronizálási szolgáltatást.
