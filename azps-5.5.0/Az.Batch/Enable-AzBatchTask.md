# Enable-AzBatchTask

## SYNOPSIS
Tevékenység újraaktiválása.

## SZINTAXIS

### Azonosító
```
Enable-AzBatchTask [-JobId] <String> [-Id] <String> -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject
```
Enable-AzBatchTask [-Task] <PSCloudTask> -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## LEÍRÁS
Az **Enable-AzBatchTask** parancsmag újraaktivál egy feladatot.
Ha egy feladat elhasználta az újrapróbálkozási számát, ez a parancsmag ennek ellenére lehetővé teszi a futtatását.

## PÉLDÁK

### 1. példa: Tevékenység újraaktiválása
```
PS C:\>Enable-AzBatchTask -JobId "Job7" -Id "Task2" -BatchContext $Context
```

Ez a parancs újraaktiválja a Feladat2 feladatot a Job7 feladatban.

### 2. példa: Tevékenység újraaktiválása a folyamat használatával
```
PS C:\>Get-AzBatchTask -JobId "Job8" -Id "Task3" -BatchContext $Context | Enable-AzBatchTask -BatchContext $Context
```

Ez a parancs a 8-as azonosítójú feladathoz az azonosító feladat3 azonosítóját használó kötegfeladatot kapja meg a Get-AzBatchTask parancsmag használatával.
A parancs a folyamat műveleti operátorával átadja a feladatot az aktuális parancsmagnak.
A parancs újraaktiválja a feladatot.

## PARAMETERS

### -BatchContext
Azt a **BatchAccountContext példányt** adja meg, amely a parancsmagot használja a Batch szolgáltatással való interakcióhoz.
Ha a Get-AzBatchAccount parancsmaggal szerezze be a BatchAccountContext parancsmagot, akkor az Azure Active Directory-hitelesítést fogja használni a Batch szolgáltatás használata során. Ha ehelyett megosztott kulcsú hitelesítést használ, a Get-AzBatchAccountKey parancsmaggal töltse ki a BatchAccountContext objektumot a hozzáférési kulcsokkal. Megosztott kulcsú hitelesítés használata esetén a rendszer alapértelmezés szerint az elsődleges hozzáférési kulcsot használja. A használni használt kulcs módosítása a BatchAccountContext.KeyInUse tulajdonsággal.
