# New-AzDataLakeAnalyticsComputePolicy

## SYNOPSIS
Adattavak-elemzési számítási házirendszabályt hoz létre egy adott AAD-entitáshoz.

## SZINTAXIS

```
New-AzDataLakeAnalyticsComputePolicy [-ResourceGroupName <String>] [-Account] <String> [-Name] <String>
 [-ObjectId] <Guid> [-ObjectType] <String> [-MaxAnalyticsUnitsPerJob <Int32>] [-MinPriorityPerJob <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## LEÍRÁS
A **New-AzDataLakeAnalyticsComputePolicy** létrehozza a megadott számítási házirendszabályt egy Adott AAD-entitáshoz egy Azure Data Lake Analytics-fiókban.

## PÉLDÁK

### 1. példa: Számítási házirend létrehozása egyetlen s szabálysal
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5
```

Ez a parancs létrehoz egy "myPolicy" nevű házirendet a "contosoadla" fiókban a "83cb7ad2-3523-4b82-b909-d478b0d8aea3" azonosítójú felhasználóhoz, amely biztosítja, hogy 5-től több elemzési egységben ne tudjanak feladatot be nyújtani.

### 2. példa: Számítási házirend létrehozása mindkét szabály beállításával
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5 -MinPriorityPerJob 100
```

Ez a parancs létrehoz egy "myPolicy" nevű házirendet a "contosoadla" fiókban a "83cb7ad2-3523-4b82-b909-d478b0d8aea3" azonosítójú felhasználóhoz, amely biztosítja, hogy ne tudjanak 5-5 elemzési egységnél több vagy 100-asnál kisebb prioritású feladatot be nyújtani.
