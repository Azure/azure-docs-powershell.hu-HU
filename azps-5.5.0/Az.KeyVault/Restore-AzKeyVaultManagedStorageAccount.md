# Restore-AzKeyVaultManagedStorageAccount

## SYNOPSIS
Visszaállít egy felügyelt tárfiókot egy kulcstárolóban egy biztonsági másolatból.

## SZINTAXIS

### ByVaultName (alapértelmezett)
```
Restore-AzKeyVaultManagedStorageAccount [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultManagedStorageAccount [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Restore-AzKeyVaultManagedStorageAccount [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## LEÍRÁS
A **Restore-AzKeyVaultManagedStorageAccount** parancsmag egy biztonsági másolatból létrehoz egy felügyelt tárfiókot a megadott kulcstárolóban.
Ez a felügyelt tárfiók a bemeneti fájlban található biztonsági másolat másolata, és a neve megegyezik az eredetivel.
Ha a kulcstároló már tartalmaz azonos nevű felügyelt tárfiókot, ez a parancsmag nem tudja felülírni az eredetit.
Az a kulcstároló, amelybe visszaállítja a felügyelt tárfiókot, eltérhet attól a kulcstártól, amelyről a felügyelt tárfiókról biztonsági mentést tárolt.
A kulcstárnak azonban ugyanazt az előfizetést kell használnia, és ugyanabban a földrajzi régióban (például Észak-Amerikában) kell lennie egy Azure-régióban.
Az Azure-régiók földrajzokkal való megfeleltetését a Microsoft Azure Trust Center https://azure.microsoft.com/support/trust-center/) webhelyén tudja meg.
