## LEÍRÁS
A **New-AzNetworkInterface** parancsmag létrehoz egy Azure hálózati felületet.

## PÉLDÁK

### 1. példa: Azure hálózati felület létrehozása
```powershell
PS C:\>New-AzNetworkInterface -Name "NetworkInterface1" -ResourceGroupName "ResourceGroup1" -Location "centralus" -SubnetId "/subscriptions/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/resourceGroups/ResourceGroup1/providers/Microsoft.Network/virtualNetworks/VirtualNetwork1/subnets/Subnet1" -IpConfigurationName "IPConfiguration1" -DnsServer "8.8.8.8", "8.8.4.4"
```

Ez a parancs létrehoz egy NetworkInterface001 nevű hálózati felületet egy dinamikusan hozzárendelt személyes IP-címmel az Alhálózat1 szolgáltatásból a VirtualNetwork1 virtuális hálózatban. A parancs két DNS-kiszolgálót is hozzárendel a hálózati felülethez. Az IPConfiguration gyermekerőforrás automatikusan létrejön az IPConfiguration1 névvel.

### 2. példa: Azure hálózati felület létrehozása IP-konfigurációs objektum használatával
```powershell
PS C:\>$Subnet = Get-AzVirtualNetwork -Name "VirtualNetwork1" -ResourceGroupName "ResourceGroup1" 
PS C:\>$IPconfig = New-AzNetworkInterfaceIpConfig -Name "IPConfig1" -PrivateIpAddressVersion IPv4 -PrivateIpAddress "10.0.1.10" -SubnetId $Subnet.Subnets[0].Id
PS C:\> New-AzNetworkInterface -Name "NetworkInterface1" -ResourceGroupName "ResourceGroup1" -Location "centralus" -IpConfiguration $IPconfig
```

Ebben a példában egy IP-konfigurációs objektum használatával új hálózati felületet hoz létre. Az IP konfigurációs objektum statikus privát IPv4-címet ad meg.
Az első parancs beolvassa a második parancsban az alhálózat hozzárendeléshez használt meglévő virtuális hálózatot.
A második parancs létrehoz egy IPConfig1 nevű hálózatifelület-IP-konfigurációt, és az IPConfig1 nevű változóban tárolja a $IPconfig.
A harmadik parancs létrehoz egy NetworkInterface1 nevű hálózati felületet, amely a hálózati felület IP-konfigurációját használja a $IPconfig.

### 3. példa

Hálózati felületet hoz létre. (automatikusan generált)
