## LEÍRÁS
A **New-AzSentinelAlertRule** parancsmag létrehoz egy analytic (riasztási szabály) a megadott munkaterületen.
A létrehozni kívánt riasztási szabály fajtáját meg kell adnia az egyik paraméternek, például az -AzureActiveDirectorynak.  Minden egyes fajtához különböző kötelező paramétereket kell megadni.
A Confirm *paraméter* és a Windows PowerShell$ConfirmPreference változó segítségével szabályozhatja, hogy a parancsmag megerősítést kér-e.
Megjegyzés: A portálon nem minden adatkapcsolat érhető el API-n keresztül.

## PÉLDÁK

### 1. példa
```powershell
PS C:\> $DataConnector = New-AzSentinelDataConnector -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -AzureSecurityCenter -Alerts Enabled -SubscriptionId ((Get-AzContext).Subscription.Id)
```

Ebben a példában létrehoz egy **DataConnectort** az Azure Biztonsági központhoz a megadott munkaterületen, majd azt egy másik $DataConnector tárolja.

### 2. példa
```powershell
PS C:\> $DataConnector = New-AzSentinelDataConnector -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -MicrosoftCloudAppSecurity -Alerts Enabled -DiscoveryLogs Disabled
```

Ebben a példában létrehoz egy **DataConnectort** a Microsoft Cloud App Security szolgáltatáshoz a megadott munkaterületen, majd azt a $DataConnector tárolja.
