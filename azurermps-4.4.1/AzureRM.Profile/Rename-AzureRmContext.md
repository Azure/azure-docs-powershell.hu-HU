# Rename-AzureRmContext

## Áttekintés
Azure-környezet átnevezése  Alapértelmezés szerint a kontextusokat felhasználói fiók és előfizetés szerint nevezi el a program.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SZINTAXISA

### Beviteli objektum (alapértelmezett)
```
Rename-AzureRmContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### Környezeti név
```
Rename-AzureRmContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## Leírás
Azure-környezet átnevezése  Alapértelmezés szerint a kontextusokat felhasználói fiók és előfizetés szerint nevezi el a program.

## Példák

### Környezet átnevezése névvel ellátott paraméterekkel
```
PS C:\> Rename-AzureRmContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

A " user1@contoso.org '" előfizetés "12345-6789-2345-3567890" "munka" nevű környezetének átnevezése  A parancs elvégzése után a környezetet a "Select-AzureRmContext" munkával is megtalálhatja.  Figyelje meg, hogy a TAB billentyűvel a "SourceName" értékek között a TAB billentyűvel válthat.

### Környezet átnevezése az elhelyezési paraméterekkel
```
PS C:\> Rename-AzureRmContext "My context" "Work"
```

Nevezze át a "saját környezet" nevű környezetet a "munka" értékre.  A parancs elvégzése után Select-AzureRmContext-munkakörnyezettel megcélzott környezete
